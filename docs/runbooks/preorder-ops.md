# Runbook: Операционный процесс предзаказов (закупщик)

## Цель
Регламент запуска волн офферов, перераспределения и завершения цикла предзаказов.

## Когда использовать
- После фактического прихода партии и фиксации цены.
- При отказах/истечениях офферов и появлении освобождённого объёма.

## Шаг 1. Подготовка данных
1. Убедиться, что по товару есть `intent_created`.
2. Зафиксировать:
   - `product_id`
   - доступный объём (`available_boxes`)
   - цена за коробку (`price_per_box`)

## Шаг 2. Запуск волны офферов (FIFO)
```bash
php bin/preorder_send_offers.php <product_id> <available_boxes> <price_per_box> [ttl_hours=4]
```
Ожидаемый результат: строка `Offers sent: ...; allocated_boxes: ...`.

## Шаг 3. Напоминания за 60 минут до истечения
Запускается по cron:
```bash
php bin/preorder_remind_expiring.php
```
Скрипт отправляет SMS и печатает `sent/failed`.

## Шаг 4. Истечение офферов
Запускается по cron:
```bash
php bin/preorder_expire_offers.php
```
Ожидаемый результат: количество переведённых в `expired`.

## Шаг 5. Перераспределение освобождённого объёма
Если после `declined/expired` есть свободный объём:
```bash
php bin/preorder_reallocate.php <product_id> <freed_boxes> [ttl_hours=4]
```
Ожидаемый результат: строка `Reallocated offers: ...`.

## Частота cron (рекомендация)
- `preorder_remind_expiring.php` — каждые 10 минут.
- `preorder_expire_offers.php` — каждые 5–10 минут.
- `preorder_reallocate.php` — вручную по событию или автоматизацией после expire/decline.

## Контрольный чек-лист закупщика
- [ ] Офферы отправлены по FIFO.
- [ ] Цена оффера соответствует фактической закупке.
- [ ] Напоминания и истечения отрабатывают.
- [ ] Освобождённый объём перераспределён.
