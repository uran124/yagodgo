# План следующей итерации (1 страница)

## Цель итерации

Снизить риск прод-инцидентов при изменениях БД и ускорить онбординг нового разработчика до “первого успешного запуска + первый PR” за 1 рабочий день.

## Результаты итерации (target outcomes)

1. Время онбординга нового разработчика: не более 4 часов до локального запуска и прохождения тестов.
2. Нулевая доля ручных миграций на окружениях (все изменения через `bin/migrate.php`).
3. Для каждого релиза: обязательный CI green перед merge.

## Backlog (приоритет сверху вниз)

1. **DX/Docs hardening**
   - Добавить в README раздел “Zero-to-run за 15 минут”.
   - Добавить troubleshooting по типовым ошибкам БД и конфигов.
2. **Migration safety**
   - Шаблон для SQL-миграций (чеклист: idempotency, locks, rollback note).
   - Pre-merge dry-run миграций на тестовой БД.
3. **CI observability**
   - Явный summary шага миграций в CI.
   - Публикация junit + короткого отчёта по упавшим тестам.
4. **Routing maintainability**
   - Выделить соглашение по структуре route-файлов (порядок, нейминг, guards).

## Объём / вне объёма

### In scope

- Документация onboarding/runbook/ADR.
- Улучшения CI вокруг миграций и тестов.
- Небольшие refactor-правки роутинга без изменения поведения.

### Out of scope

- Полная замена роутера на framework-компонент.
- Переезд конфигурации на новую платформу секретов.
- Крупные бизнес-фичи каталога/чекаута.

## Риски и меры

- **Риск:** миграция блокирует таблицу слишком долго.
  - **Мера:** проверка на тестовой копии + окно деплоя с rollback-планом.
- **Риск:** документация устаревает.
  - **Мера:** правило “любой change в деплой/миграции = update docs в том же PR”.
- **Риск:** CI нестабилен из-за среды.
  - **Мера:** health-check БД + артефакты логов + ретраи только для инфраструктурных шагов.

## Критерии готовности (DoD)

- Новый разработчик поднимает проект с нуля, следуя README, без устных пояснений.
- Есть актуальный runbook на случай падения деплоя/миграций.
- Есть ADR с обоснованием принятых инфраструктурных решений.
- Есть согласованный 1-page план следующей итерации.
